# PT.2 Theming your webbed site
<br/>
- Before changes were made to GitHub Pages interface, it was possible to pick a theme using a built-in GUI
- Currently we need to set the theme using a _**`_config.yml`**_ file. This will tell _**`Jekyll`**_ how to transform the text in the README file and generate a styled website.
- Select _**`Add File > Create New File`**_
<br/>
<ImageView image = "/src/assets/blog/github/github-add-file.png" altText="Create a new File"></ImageView>
<br/>
- On the new page opened under _**`Name your file...`**_ enter _**`_config.yml`**_
- Add the following contents to _**`_config.yml file`**_
<br/>
```sh title="_config.yml" showLineNumbers
title: Your Name
theme: jekyll-theme-minimal
```
<br/>
- A full list of supported themes can be found [here](https://pages.github.com/themes/)
- Save your changes by _**`Committing changes...`**_
<br/>
<ImageView image = "/src/assets/blog/github/github-config-yaml.png" altText="_config.yml file"></ImageView>
<br/>
- A build should be automatically triggered
- Progress can be tracked under the repository's _**`Actions`**_ tab
<br/>
<ImageView image = "/src/assets/blog/github/github-actions-tab.png" altText="GitHub Actions tab"></ImageView>
<br/>
- Once deployment is done, the site we created from the previous <a href="/blog/2025-06-23-host-your-own-site-part-1-using-github" aria-label="Visit the blog page">PT.1 Webbed Site Repository</a> should now be themed
